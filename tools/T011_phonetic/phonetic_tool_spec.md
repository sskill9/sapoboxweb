# フォネティックコード表示ツール 仕様書

## 1. 概要
本ツールは、コールセンター業務においてアルファベット・数字・記号を電話口で正確に伝えるため、
各文字に対応するフォネティックコード（言い換え表現）を即座に確認できる支援ツールである。

入力した文字列の各文字にマウスカーソルを合わせることで、その文字に対するフォネティックコード候補を表示する。

## 2. 想定利用シーン
- 会員ID、受付番号、シリアルキーの読み上げ
- メールアドレス、URL、ユーザー名の口頭案内
- 記号を含む文字列の聞き間違い防止

## 3. 基本仕様

### 3.1 入力
- 1行テキスト入力
- 対応文字
  - アルファベット（A–Z / a–z）
  - 数字（0–9）
  - 各種記号（@ . - _ / \ # : ; + * = ? ! & % $ ¥ () [] {} <> " ' , ~ ^ |）

### 3.2 大文字・小文字の扱い
- 大文字・小文字は区別しない
- 必要に応じてオペレーターが口頭で
  「大文字の」「小文字の」と補足する運用とする

### 3.3 表示方式
- 文字ホバー時にツールチップで表示
- 同時に補助表示エリアへも内容を表示

## 4. フォネティックコード仕様

### 4.1 基本方針
- NATOフォネティックアルファベットは不採用
- 日本の一般利用者が理解しやすい「身近なもの」を優先
- 「〇〇のA」形式で統一
- 1文字につき最大3候補
- 表示順は「伝わりやすい順」

### 4.2 データ定義形式
- JSONは使用せず、JavaScriptファイルで定義（CORS回避）
- 定義例：

```js
const PHONETIC_MAP = {
  "A": ["アメリカのA", "アップルのA", "アルファベットのA"],
  "1": ["いちの1", "ワンの1", "ひとつの1"],
  "@": ["アットマークの@", "メールアドレスの途中にある@（アット）", "アットの@"]
};
```

- 参照時は `toUpperCase()` を使用し、大文字キーで統一

## 5. 非機能要件
- 完全ローカル動作
- 外部通信なし
- 単体ZIP配布可能
- PC利用前提（マウスホバー操作）

## 6. 運用ルール
- 表示される候補は、通じない場合に順に言い換える
- 1候補目で通じない場合、2候補目、3候補目を使用する

## 7. 将来拡張（予定）
- 文字列全体を読み上げ文に変換する機能
- クリックによるコピー機能
- スマートフォン対応（タップ操作）
